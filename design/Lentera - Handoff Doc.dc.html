Lentera — Spesifikasi Sistem

Aplikasi terapi memori personal: mencatat interaksi harian dengan orang sekitar untuk menjaga rasa syukur (kebaikan) sekaligus batasan diri (hal negatif), dengan privasi zero-knowledge sebagai harga mati.

Tiga komponen: Mobile (Flutter) · API Backend (Laravel) · Web Konsol (React). Dokumen ini adalah cetak biru desain & arsitektur — prototipe interaktif tersedia terpisah.

1 · Alur Pengguna (catat <30 detik)

Jalur tercepat dari membuka app hingga momen tersimpan & terenkripsi.

2 · Wireframe Layar Android

Enam layar inti. Pembeda visual positif/negatif: garis aksen tipis di sisi kiri kartu + ikon (💚 hati / ⚠️ perisai / 💬 chat).

3 · Skema Database (PostgreSQL)

Prinsip: kolom sensitif (nama orang, isi catatan) disimpan sebagai bytea ciphertext. Metadata non-sensitif (tipe, tanggal) tetap plaintext agar server bisa menghitung statistik tanpa pernah membaca isi.

4 · Alur Enkripsi End-to-End (Zero-Knowledge)

Kunci enkripsi tidak pernah meninggalkan device. Server hanya menyimpan ciphertext yang tidak bisa dibaca admin maupun pemilik server.

{{ c.n }} {{ c.title }}
{{ c.desc }}
Kunci: key = Argon2id(passphrase, kdf_salt) diturunkan saat login, disimpan hanya di memori (Android Keystore / sesi browser). Setiap record pakai nonce acak 96-bit. Algoritma: AES-256-GCM (authenticated encryption → sekaligus deteksi tampering).
